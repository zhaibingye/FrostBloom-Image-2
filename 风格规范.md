# DeepSeek Chat 前端设计风格规范

## 1. 设计定位

DeepSeek Chat 的界面风格是「极简、轻盈、圆润、低干扰」的 AI 对话产品风格。整体以大面积白色留白承载内容，用高饱和蓝色作为品牌识别和主要交互反馈，辅以极浅边框、柔和阴影和胶囊形控件，营造清爽、可信、专注输入的体验。

关键词：白底留白、品牌蓝、圆角胶囊、轻边框、柔和阴影、中心聚焦、低密度信息。

## 2. 色彩规范

### 2.1 主色

| 用途 | 颜色 | 说明 |
| --- | --- | --- |
| 品牌主蓝 | `#3964FE` | Logo、选中态文字、主要图标、主按钮背景 |
| 选中浅蓝底 | `#EDF3FE` | 分段控件选中态、功能按钮选中态 |
| 选中描边蓝 | `#B7C8FE` | 选中态描边、选中态投影描边 |
| 焦点蓝 | `#3B82F6` | 键盘焦点环 |

### 2.2 中性色

| 用途 | 颜色 | 说明 |
| --- | --- | --- |
| 页面背景 | `#FFFFFF` | 主内容区背景 |
| 侧边栏背景 | `#F9FAFB` | 展开侧边栏背景 |
| 主文字 | `#0F1115` | 标题、按钮文字、正文输入 |
| 次级文字 | `#81858C` | 空状态、弱提示、次级图标 |
| 输入占位 | 近似 `#A9B2C0` | 文本框 placeholder，需保持低对比但可读 |
| 轻边框 | `rgba(0, 0, 0, 0.1)` | 卡片、按钮、分段控件边框 |
| 遮罩 | `rgba(0, 0, 0, 0.4)` | 侧边栏或弹层背景遮罩 |

### 2.3 用色原则

- 页面默认只使用白、黑、灰、蓝四类颜色，避免引入多余强调色。
- 蓝色仅用于品牌、选中态、主操作和正向高亮，不用于普通文本。
- 边框尽量使用 10% 黑色透明度，避免重描边。
- 浅蓝背景必须配合蓝色文字或蓝色描边，形成明确选中语义。

## 3. 字体规范

### 3.1 字体族

```css
font-family: quote-cjk-patch, Inter, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen, Ubuntu, Cantarell, "Open Sans", "Helvetica Neue", sans-serif;
```

### 3.2 字号与字重

| 场景 | 桌面端 | 移动端 | 字重 | 行高 |
| --- | --- | --- | --- | --- |
| 欢迎标题 | `24px` | `20px` | `600` 桌面 / `500` 移动 | `32px` 桌面 / `28px` 移动 |
| 输入正文 | `16px` | `16px` | `400` | `24px` |
| 分段控件文字 | `14px` | `14px` | `500` | `16.8px` |
| 功能按钮文字 | `13px` | `13px` | `500` | `24px` |
| 侧边栏按钮文字 | `14px` | `14px` | `500` | `25px` |
| 空状态文字 | `12px` | `12px` | `500` | `18px` |

### 3.3 排版原则

- 中文标题不使用超大字号，依靠居中、图标和留白建立视觉焦点。
- 字重整体克制，主要在 `400` 到 `600` 之间。
- 按钮文字使用中等字重，避免粗重按钮造成压迫感。

## 4. 布局规范

### 4.1 整体结构

- 主页面以水平 Flex 布局组织：左侧可展开侧边栏，右侧主对话区。
- 初始空白页将欢迎区与输入框置于页面中上部，视觉中心略高于屏幕正中。
- 桌面端主内容容器最大宽度约 `840px`，输入框宽度约 `776px`。
- 移动端仍保留居中主内容，但内容宽度可超过视口布局基准，实际可见区以横向居中裁切呈现。

### 4.2 顶部操作区

- 桌面端：左上角显示品牌图标和一个胶囊形操作组，尺寸约 `76px * 40px`。
- 移动端：顶部栏高度约 `60px`，左右各一个 `44px` 圆形图标按钮。
- 顶部按钮不使用实底，默认透明，仅保留黑色图标。

### 4.3 侧边栏

- 展开宽度约 `261px`。
- 背景为 `#F9FAFB`。
- 内边距约 `6px 12px 10px`。
- Logo 位于顶部，品牌蓝显示。
- 新对话按钮位于顶部下方，宽度约 `236px`，高度 `40px`。
- 历史为空时使用居中空状态，灰色图标和 `12px` 文案。
- 用户信息固定在底部附近，头像 `32px` 圆形，用户行圆角 `12px`。

| 图标按钮尺寸 | `34px` 桌面 / `44px` 移动顶部 |

## 6. 圆角规范

| 组件 | 圆角 |
| --- | --- |
| 输入面板 | `24px` |
| 分段控件容器 | `120px` |
| 分段控件选中滑块 | `120px` |
| 功能开关按钮 | `18px` |
| 主发送按钮 | `50%` |
| 图标按钮 | `50%` |
| 侧边栏新对话按钮 | `100px` |

| 头像 | `50%` |

设计原则：能做成胶囊的控件尽量胶囊化；面板使用大圆角；小图标按钮使用正圆。

## 7. 阴影与边框

### 7.1 边框

- 默认细边框：`0.8px solid rgba(0, 0, 0, 0.1)`。
- 选中按钮边框：`0.8px solid #B7C8FE`。
- 侧边栏主按钮边框默认透明：`0.8px solid rgba(103, 158, 254, 0)`。

### 7.2 阴影

输入面板阴影：

```css
box-shadow:
  rgba(0, 0, 0, 0.02) 0 4px 12px,
  rgba(72, 104, 178, 0.01) 0 2px 2px,
  rgba(72, 104, 178, 0.03) 0 30px 60px;
```

分段控件选中态阴影：

```css
box-shadow:
  #B7C8FE 0 0 0 1px,
  rgba(0, 0, 0, 0.04) 0 4px 8px;
```

侧边栏新对话按钮阴影：

```css
box-shadow:
  rgba(72, 104, 178, 0.04) 0 -2px 2px,
  rgba(106, 111, 117, 0.09) 0 2px 2px,
  rgba(72, 104, 178, 0.08) 0 1px 2px;
```

设计原则：阴影应大而淡，不产生明显悬浮卡片感，只用于强化层级和可点击性。

## 8. 组件规范

### 8.1 欢迎标题

- 图标与标题横向排列、垂直居中。
- 桌面端 Logo 图标约 `34px * 25px`，移动端约 `30px * 22px`。
- 图标与文字间距约 `10px` 到 `14px`。
- 标题文案根据当前模式变化，例如“使用快速模式开始对话”。

### 8.2 模式分段控件

- 容器尺寸约 `286px * 38px`。
- 容器背景白色，轻边框，圆角 `120px`。
- 内部两项等宽，每项约 `139px * 34px`。
- 容器内间距：`padding: 1px`，选项间距 `4px`。
- 选中态使用浅蓝底、蓝色文字、蓝色描边投影。
- 未选中态文字为主黑色，无背景。
- 每项包含 `15px` 图标和 `14px` 文案。

### 8.3 输入面板

- 桌面端宽度约 `776px`，高度约 `124px`。
- 移动端可视宽度约 `436px`，高度约 `106px`。
- 背景白色，圆角 `24px`，细边框和柔和投影。
- 文本输入区字体 `16px / 24px`。
- Placeholder 使用浅灰蓝，语气直接：“给 DeepSeek 发送消息”。
- 工具栏位于输入面板底部，左侧为功能按钮，右侧为附件与发送按钮。

### 8.4 功能开关按钮

- 高度 `34px`。
- 水平内边距 `12px`。
- 圆角 `18px`。
- 字号 `13px`，字重 `500`。
- 默认态：透明背景、黑色文字、浅灰边框。
- 选中态：`#EDF3FE` 背景、`#3964FE` 文字、`#B7C8FE` 边框。
- 图标尺寸约 `14px`。

### 8.5 发送按钮

- 尺寸 `34px * 34px`，圆形。
- 背景品牌蓝 `#3964FE`，图标白色。
- 禁用时仍保留蓝色形态，但视觉上可降低透明度或使用较浅蓝，避免误以为不可用组件消失。

### 8.6 图标按钮

- 桌面常规尺寸 `34px * 34px`。
- 移动顶部尺寸 `44px * 44px`。
- 默认透明背景，圆形热区。
- 图标颜色默认 `#0F1115`，次级操作可用 `#81858C`。
- Hover 可使用极浅灰背景，保持低干扰。
- Focus 使用蓝色焦点环。

## 9. 响应式规范

### 9.1 桌面端

- 页面主内容居中，侧边栏可展开。
- 欢迎标题字号 `24px`，字重 `600`。
- 输入面板宽度约 `776px`。
- 顶部左侧使用品牌图标与胶囊操作组。

### 9.2 移动端

- 顶部操作栏高度 `60px`，左右按钮尺寸 `44px`。
- 欢迎标题字号降为 `20px`，字重 `500`。
- 模式切换控件宽度保持约 `286px`，居中显示。
- 输入面板在窄屏中保持大圆角与工具栏结构，避免改为全宽扁平输入框。
- 功能按钮和发送按钮仍位于同一工具栏内，保持与桌面一致的心智模型。

## 10. 交互状态

| 状态 | 表现 |
| --- | --- |
| 默认 | 白底、浅边框、黑色文字 |
| Hover | 图标按钮出现极浅灰圆形背景；按钮边框或背景轻微加强 |
| Selected | 浅蓝底、蓝色文字、蓝色描边或投影 |
| Focus | 蓝色焦点环，圆角与组件一致 |
| Disabled | 保留控件形状，降低操作暗示，不移除视觉位置 |
| Overlay | 使用 `rgba(0, 0, 0, 0.4)` 遮罩 |

## 11. 图标风格

- 使用线性图标为主，线条简洁，圆润端点。
- 品牌 Logo 使用实心蓝色图形。
- 功能图标尺寸集中在 `14px`、`15px`、`16px`、`20px` 四档。
- 图标颜色继承文字色，选中态统一变为品牌蓝。

## 13. 设计落地注意事项

- 不要过度增加卡片、分割线和装饰图形，DeepSeek 风格依赖留白而不是复杂布局。
- 不要使用大面积渐变或彩色背景，会破坏清爽的工具产品气质。
- 按钮不要做厚重阴影，阴影透明度需低，模糊半径可大。
- 圆角要统一，输入框、分段控件、按钮都应呈现柔和、胶囊化语言。
- 主要交互控件应始终围绕输入框组织，突出“对话输入”这一核心任务。
